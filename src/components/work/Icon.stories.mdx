# Icon

### Icons are visual symbols used to represent ideas, objects, or actions. They communicate messages at a glance, afford interactivity, and draw attention to important information.

<Preview withToolbar>
  <Story name="Standard">

## Usage

All Icons are sourced from those available in our **[Icon Library](/?path=/docs/application-libraries-icon-library--app-icons)**.

Icons are helpful by:

- Referencing an existing content type within application, for example a type of issue or template.
- Helping users understand UI elements and navigate an interface
- Communicating a secondary status, for example the priority of an issue.

When you use an Icon, consider where and how it is currently used elsewhere in the Lightbox suite of applications, and make sure that the meaning you prescribe to it is the same or at least similar. Don't repurpose an established Icon for a different action.

Be mindful of how Icons are used in specific components, like a **[Command](/?path=/docs/atoms-command--documentation)** or a **[Badge](/?path=/docs/atoms-badge--documentation)**.

## Principles

- **Additive:**
  By and large, Icons should be used with text.

- **Specific:**
  Icons should have specific meaning, and be used to relay that meaning.

- **Functional:**
  Icons should be in service of a user action. Icons should not be decorative.
